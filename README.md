# demo-storage-jpa

## 어노테이션 가이드

도메인 객체에 Storage 어노테이션을 사용하면 빌드 시 JPA Entity가 자동 생성됩니다.

### @StorageTable

클래스에 적용하여 테이블 매핑을 정의합니다.

```java
@StorageTable("orders")  // 테이블명 지정
public class Order {
    // ...
}

@StorageTable  // 테이블명 미지정 시 클래스명을 snake_case로 변환 (Order → order)
public class Order {
    // ...
}
```

### @StorageId

기본키 필드를 지정합니다.

```java
@StorageId  // autoGenerated = true (기본값), IDENTITY 전략 사용
private Long id;

@StorageId(autoGenerated = false)  // 직접 ID 할당
private String id;
```

### @StorageColumn

컬럼 매핑을 정의합니다.

```java
@StorageColumn(value = "order_no", nullable = false, unique = true)
private String orderNumber;

@StorageColumn(nullable = false)  // 컬럼명 미지정 시 필드명을 snake_case로 변환
private String customerName;

@StorageColumn  // 기본값: nullable = true, unique = false
private String description;
```

### @StorageEnum

Enum 타입 필드의 저장 방식을 지정합니다.

```java
@StorageEnum(EnumType.STRING)  // VARCHAR로 저장 (권장)
private OrderStatus status;

@StorageEnum(EnumType.ORDINAL)  // 정수로 저장
private Priority priority;
```

### @StorageRelation

연관관계를 정의합니다.

```java
// ManyToOne
@StorageRelation(type = RelationType.MANY_TO_ONE)
private Order order;

// OneToMany (양방향)
@StorageRelation(type = RelationType.ONE_TO_MANY, mappedBy = "order")
private List<OrderItem> items;

// OneToOne
@StorageRelation(type = RelationType.ONE_TO_ONE)
private OrderDetail detail;

// ManyToMany
@StorageRelation(type = RelationType.MANY_TO_MANY)
private List<Tag> tags;
```

### @StorageTransient

Entity에서 제외할 필드를 지정합니다.

```java
@StorageTransient  // Entity 필드에서 제외됨
private String tempCalculation;
```

## 자동 코드 생성

### 입력 (도메인 객체)

```java
package com.example.domain;

@StorageTable("orders")
public class Order {

    @StorageId
    private Long id;

    @StorageColumn(value = "order_no", nullable = false, unique = true)
    private String orderNumber;

    @StorageColumn(nullable = false)
    private String customerName;

    @StorageEnum(EnumType.STRING)
    private OrderStatus status;

    @StorageRelation(type = RelationType.ONE_TO_MANY, mappedBy = "order")
    private List<OrderItem> items;

    @StorageTransient
    private String tempCalculation;

    // getter, setter...
}
```

### 출력 (자동 생성)

빌드 시 다음 파일들이 자동 생성됩니다:

```
build/generated/sources/annotationProcessor/java/main/
└── com/example/domain/
    ├── entity/
    │   └── OrderEntity.java
    ├── mapper/
    │   └── OrderStorageMapper.java
    └── repository/
        └── OrderEntityRepository.java
```

#### OrderEntity.java

```java
@Entity
@Table(name = "orders")
public class OrderEntity {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "order_no", nullable = false, unique = true)
    private String orderNumber;

    @Column(name = "customer_name", nullable = false)
    private String customerName;

    @Column(name = "status")
    @Enumerated(EnumType.STRING)
    private OrderStatus status;

    @OneToMany(mappedBy = "order")
    private List<OrderItemEntity> items;

    // 기본 생성자, getter, setter...
}
```

#### OrderStorageMapper.java

```java
@Component
public class OrderStorageMapper implements DomainMapper<Order, OrderEntity> {

    @Override
    public OrderEntity toStorage(Order domain) {
        if (domain == null) return null;
        OrderEntity entity = new OrderEntity();
        entity.setId(domain.getId());
        entity.setOrderNumber(domain.getOrderNumber());
        entity.setCustomerName(domain.getCustomerName());
        entity.setStatus(domain.getStatus());
        return entity;
    }

    @Override
    public Order toDomain(OrderEntity entity) {
        if (entity == null) return null;
        Order domain = new Order();
        domain.setId(entity.getId());
        domain.setOrderNumber(entity.getOrderNumber());
        domain.setCustomerName(entity.getCustomerName());
        domain.setStatus(entity.getStatus());
        return domain;
    }
}
```

#### OrderEntityRepository.java

```java
public interface OrderEntityRepository extends JpaRepository<OrderEntity, Long> {
}
```

## Adapter 사용법

### 방법 1: AutoJpaAdapter (권장)

Repository와 Mapper를 자동으로 연결합니다. 예외 변환 기능이 내장되어 있습니다.

```java
@Adapter
public class OrderJpaAdapter extends AutoJpaAdapter<Order, Long> 
        implements OrderPort {

    public OrderJpaAdapter(ApplicationContext context) {
        super(Order.class, context);
    }

    @Override
    public Order save(Order order) {
        return saveWithException(order);  // 예외 자동 변환
    }

    @Override
    public Optional<Order> findById(Long id) {
        return findByIdWithException(id);
    }

    @Override
    public Order findByIdOrThrow(Long id) {
        return findByIdOrThrow(id);  // 없으면 StorageException 발생
    }

    @Override
    public List<Order> findAll() {
        return findAllWithException();
    }

    @Override
    public void delete(Order order) {
        deleteWithException(order);
    }
}
```

#### AutoJpaAdapter 제공 메서드

| 메서드 | 예외 변환 | 설명 |
|--------|----------|------|
| `save(D)` | ❌ | 저장 |
| `saveWithException(D)` | ✅ | 저장 (예외 변환) |
| `saveAll(List<D>)` | ❌ | 일괄 저장 |
| `saveAllWithException(List<D>)` | ✅ | 일괄 저장 (예외 변환) |
| `findById(ID)` | ❌ | ID로 조회 |
| `findByIdWithException(ID)` | ✅ | ID로 조회 (예외 변환) |
| `findByIdOrThrow(ID)` | ✅ | ID로 조회, 없으면 예외 |
| `findAll()` | ❌ | 전체 조회 |
| `findAllWithException()` | ✅ | 전체 조회 (예외 변환) |
| `delete(D)` | ❌ | 삭제 |
| `deleteWithException(D)` | ✅ | 삭제 (예외 변환) |
| `deleteById(ID)` | ❌ | ID로 삭제 |
| `deleteByIdWithException(ID)` | ✅ | ID로 삭제 (예외 변환) |
| `existsById(ID)` | ❌ | 존재 여부 확인 |
| `count()` | ❌ | 개수 조회 |

### 방법 2: ManualJpaAdapter

Repository와 Mapper를 직접 주입받습니다. 타입 안전성이 보장됩니다.

```java
@Adapter
public class OrderJpaAdapter 
        extends ManualJpaAdapter<Order, OrderEntity, Long, OrderEntityRepository>
        implements OrderPort {

    public OrderJpaAdapter(OrderEntityRepository repository, 
                           OrderStorageMapper mapper) {
        super(repository, mapper);
    }

    @Override
    public Order save(Order order) {
        return save(order);
    }

    @Override
    public Optional<Order> findById(Long id) {
        return findById(id);
    }
}
```

#### Function 방식 (Mapper 없이)

```java
@Adapter
public class OrderJpaAdapter 
        extends ManualJpaAdapter<Order, OrderEntity, Long, OrderEntityRepository>
        implements OrderPort {

    public OrderJpaAdapter(OrderEntityRepository repository) {
        super(repository);  // Mapper 없이 생성
    }

    @Override
    public Order save(Order order) {
        return save(order, 
            OrderEntity::from,      // Domain → Entity 변환
            OrderEntity::toDomain   // Entity → Domain 변환
        );
    }
}
```

## 예외 처리

### StorageException

영속성 계층 예외를 도메인 예외로 변환합니다.

```java
// 정적 팩토리 메서드
StorageException.of("저장 실패");
StorageException.of("저장 실패", cause);
StorageException.saveFailed("Order", cause);
StorageException.notFound("Order", 1L);
StorageException.deleteFailed("Order", cause);
StorageException.duplicate("Order", "orderNumber", "ORD-001");
```

### 에러 코드

| 코드 | 설명 |
|------|------|
| `STORAGE_SAVE_FAILED` | 저장 실패 |
| `STORAGE_NOT_FOUND` | 조회 실패 (데이터 없음) |
| `STORAGE_DELETE_FAILED` | 삭제 실패 |
| `STORAGE_DUPLICATE` | 중복 데이터 |
| `STORAGE_ERROR` | 일반 오류 |

### 사용 예시

```java
@Adapter
public class OrderJpaAdapter extends AutoJpaAdapter<Order, Long> {

    // ... 생성자

    public Order findByOrderNumber(String orderNumber) {
        return repository.findByOrderNumber(orderNumber)
            .map(mapper::toDomain)
            .orElseThrow(() -> StorageException.notFound("Order", orderNumber));
    }

    public Order saveWithDuplicateCheck(Order order) {
        try {
            return saveWithException(order);
        } catch (DataIntegrityViolationException e) {
            throw StorageException.duplicate("Order", "orderNumber", order.getOrderNumber());
        }
    }
}
```

## QueryDSL 지원

### QueryDslAdapter

복잡한 쿼리가 필요할 때 QueryDSL을 사용합니다.

```java
@Adapter
public class OrderQueryAdapter extends QueryDslAdapter {

    private final QOrderEntity order = QOrderEntity.orderEntity;
    private final OrderStorageMapper mapper;

    public OrderQueryAdapter(JPAQueryFactory queryFactory, 
                             OrderStorageMapper mapper) {
        super(queryFactory);
        this.mapper = mapper;
    }

    public List<Order> findByStatus(OrderStatus status) {
        return queryFactory
            .selectFrom(order)
            .where(order.status.eq(status))
            .orderBy(order.orderedAt.desc())
            .fetch()
            .stream()
            .map(mapper::toDomain)
            .toList();
    }

    public Page<Order> findByCustomerName(String customerName, Pageable pageable) {
        List<OrderEntity> content = queryFactory
            .selectFrom(order)
            .where(order.customerName.containsIgnoreCase(customerName))
            .offset(pageable.getOffset())
            .limit(pageable.getPageSize())
            .orderBy(order.id.desc())
            .fetch();

        Long total = queryFactory
            .select(order.count())
            .from(order)
            .where(order.customerName.containsIgnoreCase(customerName))
            .fetchOne();

        List<Order> domains = content.stream()
            .map(mapper::toDomain)
            .toList();

        return toPage(domains, pageable, total);
    }
}
```

### 페이징 헬퍼

```java
// QueryDslAdapter에서 제공
protected <T> Page<T> toPage(List<T> content, Pageable pageable, Long total) {
    return new PageImpl<>(content, pageable, total != null ? total : 0L);
}
```

## 어노테이션 변환 규칙

| Storage 어노테이션 | JPA 어노테이션 | 비고 |
|-------------------|---------------|------|
| `@StorageTable` | `@Entity`, `@Table` | 테이블명 미지정 시 snake_case 변환 |
| `@StorageId` | `@Id`, `@GeneratedValue` | autoGenerated=true 시 IDENTITY 전략 |
| `@StorageColumn` | `@Column` | nullable, unique 속성 변환 |
| `@StorageEnum` | `@Enumerated` | STRING/ORDINAL 변환 |
| `@StorageRelation` | `@ManyToOne`, `@OneToMany` 등 | RelationType에 따라 변환 |
| `@StorageTransient` | (생성 제외) | Entity 필드에서 제외 |

## 주의사항

1. **도메인 객체에 getter/setter 필요**: Mapper가 getter/setter를 사용하여 값을 복사합니다.

2. **연관관계 필드는 Mapper에서 제외**: `@StorageRelation`이 붙은 필드는 자동 매핑되지 않습니다. 필요시 직접 처리하세요.

3. **@StorageTransient 필드는 Entity에서 제외**: 영속화하지 않을 필드에 사용합니다.

4. **Repository 패키지 스캔 설정**: 생성된 Repository가 스캔되도록 설정이 필요할 수 있습니다.
   ```java
   @EnableJpaRepositories(basePackages = "com.example.domain.repository")
   ```

5. **QueryDSL Q클래스**: Entity가 생성된 후 Q클래스가 생성됩니다. 빌드 순서에 주의하세요.
